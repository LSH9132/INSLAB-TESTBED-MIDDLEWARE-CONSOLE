# 실험환경 구성

이 문서는 현재 저장소의 코드 구조 설명이 아니라, 실제 테스트베드의 물리/논리 네트워크 구성을 기록하는 운영 메모입니다.

애플리케이션 실행 방법과 서비스 역할은 루트 [`README.md`](/Users/lsh/INSLAB-TESTBED-MIDDLEWARE-CONSOLE/README.md)를 기준으로 확인합니다.

## 물리 환경

- 스위치: NexTU L2 관리형 스위치
- 실험 노드: Raspberry Pi 14대
- 연결 방식: 각 Pi가 스위치 1포트에 연결

## 네트워크 구조

```text
eth0
  └── mgmt-br (10.10.0.x/24)
        |
   patch-m-t <-> patch-t-m
        |
     test2-br
        ├── int12
        ├── int14
        └── ...
```

- `mgmt-br`: SSH, 모니터링, 운영용 관리망
- `test2-br`: 실험용 VLAN 인터페이스 브리지
- `patch-m-t` / `patch-t-m`: 관리망 브리지와 실험 브리지를 잇는 patch port

## 주소 체계

- 관리망: `10.10.0.x/24`
- 실험망: `10.{낮은노드}.{높은노드}.{1|2}/30`
- 인터페이스명: `int<VLAN>`

## 핵심 운영 원칙

- 관리망과 실험망을 분리해 SSH 연결을 유지합니다.
- 링크 단위 격리는 VLAN으로 처리합니다.
- 네트워크 메트릭의 수집과 저장은 현재 `net-agent -> log-server` 경로가 담당합니다.
- `central-server`는 실험망 원시 시계열을 저장하지 않습니다.

## 참고 토폴로지 예시

자주 쓰는 인터페이스/VLAN 예시:

| Link | VLAN | Interface |
| --- | ---: | --- |
| `pi1-pi2` | `12` | `int12` |
| `pi1-pi4` | `14` | `int14` |
| `pi5-pi8` | `58` | `int58` |
| `pi6-pi9` | `69` | `int69` |
| `pi8-pi11` | `81` | `int81` |
| `pi13-pi14` | `43` | `int43` |

## 운영 체크리스트

- 각 PI의 `NODE_ID`가 `central-server` 등록값과 같은지 확인
- `log-server` TCP 포트가 외부에서 접근 가능한지 확인
- 스위치 trunk/tag 설정이 실험 토폴로지와 일치하는지 확인
- 관리망 IP와 실험망 IP가 혼선 없이 분리돼 있는지 확인
- `log-server`와 PostgreSQL, Redis가 함께 기동하는지 확인
